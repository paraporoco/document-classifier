# Document Sensitivity Classifier

Classifies documents by data sensitivity level. Runs entirely on your machine — no document content is ever sent to an external service.

**Policy version:** 1.1
**Six sensitivity levels:** PUBLIC · FOUO · CONFIDENTIAL · PERSONAL_CONFIDENTIAL · HIGHLY_CONFIDENTIAL · PERSONAL_HIGHLY_CONFIDENTIAL

---

## How it works

A TF-IDF + MLP neural network trained on labelled document examples. Inference is temperature-calibrated and returns a confidence score alongside the classification. Low-confidence results are flagged for manual review.

Two deployment targets share the same trained model:

| Target | How to run | Privacy |
|---|---|---|
| **Desktop CLI** | `classify.exe <folder>` | Local — nothing leaves the machine |
| **Web interface** | Browser SPA via `uvicorn` / Docker | Local — documents never leave the browser |

---

## Quick start

### Desktop (Windows)

```powershell
classifier_production\dist\classify.exe path\to\folder
classifier_production\dist\classify.exe path\to\document.pdf --recursive
```

Output: a CSV log and summary file alongside the input.

### Web server

```powershell
# Install dependencies (first time only)
pip install -r classifier_web\requirements-web.txt

# Generate browser-compatible model artefacts (after each retrain, or first time)
cd classifier_production
python convert_artefacts.py

# Start the server
cd ..\classifier_web
uvicorn server:app --reload
```

Open [http://localhost:8000](http://localhost:8000). Drop files onto the page or paste text. Classification runs locally in your browser — no file content is uploaded.

### Docker

```bash
# Build (from project root, after running convert_artefacts.py)
docker build -t doc-classifier-web ./classifier_web

# Run
docker run -p 8000:8000 doc-classifier-web
```

## Classification hierarchy

| Level | When to use |
|---|---|
| `PUBLIC` | Authorised for external publication. No restrictions. |
| `FOUO` | Internal use only. Routine operational content. |
| `CONFIDENTIAL` | Strategy, contracts, legal matters, financial projections. |
| `PERSONAL_CONFIDENTIAL` | Any CONFIDENTIAL content that also contains personal data (name, email, phone, DOB, ID). |
| `HIGHLY_CONFIDENTIAL` | M&A activity, credentials, board materials, security findings. |
| `PERSONAL_HIGHLY_CONFIDENTIAL` | HIGHLY_CONFIDENTIAL content that also contains personal data. |

**Key boundary rule:** a FOUO document containing any personal data is automatically `PERSONAL_CONFIDENTIAL`, not FOUO.

## Project structure

```
document_classifier/
    classifier_training/        Training pipeline (supervised + SSL)
        data/real_examples.py   Add real labelled examples here
        main.py                 Run to retrain: python main.py
        artefacts/              Training outputs (*.pkl)

    classifier_production/      Desktop deployment
        app.py                  CLI entrypoint
        convert_artefacts.py    Exports model to ONNX + JSON for the web server
        artefacts/              Copy pkl files here after retraining
        dist/classify.exe       Pre-built executable

    classifier_web/             Web deployment
        server.py               FastAPI server
        web/                    Browser SPA (no build step required)
        web_artefacts/          Generated by convert_artefacts.py (not committed)
```

## Improving the model

The model is trained on synthetic data. Adding real labelled documents substantially improves accuracy.

1. Extract text from a document (use `classify.exe` or copy manually)
2. Anonymise any personal data before adding to the training set
3. Add a `Sample(...)` entry to `classifier_training/data/real_examples.py`
4. Retrain: `cd classifier_training && python main.py`
5. Copy artefacts to production and rebuild

See [`classifier_training/retraining-procedure.md`](classifier_training/retraining-procedure.md) for the full step-by-step procedure.

## API

The web server exposes a small read-only API:

```
GET  /api/v1/health     Liveness check
GET  /api/v1/version    Model metadata and class list
POST /api/v1/classify   Server-side classify (disabled by default)
```

To enable the server-side classify endpoint:

```bash
ENABLE_SERVER_CLASSIFY=true uvicorn server:app
```

Note: this endpoint accepts `{"text": "..."}` — no file upload. Browser users always classify client-side regardless of this setting.
